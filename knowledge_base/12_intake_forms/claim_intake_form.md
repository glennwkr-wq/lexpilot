# Карта обязательных данных для претензии

## Цель

Перед подготовкой претензии LexPilot должен проверить, достаточно ли данных для заполнения шаблона `claim_generic.md`.

## Обязательные данные

- заявитель;
- адресат;
- основание правоотношений;
- дата возникновения обязательств;
- описание нарушения;
- требования заявителя;
- срок исполнения требований;
- доказательства;
- приложения.

## Дополнительные данные

- сумма требования;
- расчет задолженности, неустойки или убытков;
- реквизиты договора;
- пункт договора о претензионном порядке;
- срок ответа по договору или закону;
- банковские реквизиты для оплаты.

## Если данных не хватает

LexPilot не должен писать претензию сразу.

Нужно вывести список недостающих данных.

## Минимальный результат

Если данных достаточно, LexPilot готовит черновик претензии по `claim_generic.md`.
